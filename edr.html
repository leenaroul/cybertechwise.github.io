CrowdStrike Falcon (CQL) Hunting Queries

Process Execution Analysis

Hunts for system command line arguments running out of unusual directories like temp or public spaces.

#event_simpleName=ProcessRollup2
| regex("FilePath", g="\\(temp|Users\\Public)\\")
| table([Timestamp, ComputerName, UserName, ImageFileName, CommandLine])

Network Connection Breakdown

Identifies outgoing connection handshakes originating from native binary command lines (e.g., PowerShell, cmd.exe).

#event_simpleName=NetworkConnectIP4
| ImageFileName=/\\(powershell|cmd|wscript)\.exe$/
| stats(count=count(), dcount(RemoteIP) as DistinctIPs) by=[ComputerName, ImageFileName]

EDR Operations & Troubleshooting Procedures

Issue: Sensor Deployment Token Verification Failure

Occurs during silent command-line rollouts if the dynamic cloud installation token expires or contains configuration string flaws.

Resolution Runbook:

  • Navigate to the Falcon Platform console -> Host setup and management -> Sensor Downloads.
  • Verify if your current CID token has expired or requires a target group allocation tag.
  • When executing via Command Prompt (Admin), ensure the string quotes wrapper is pristine:
WindowsSensor.exe /install /quiet CID=YOUR_VALID_CID_HERE_WITH_NO_SPACES

Issue: Dynamic Sensor Policy De-synchronization

The host endpoints remain listed inside the dashboard container but show stale prevention posture settings or fail to pull structural logic upgrades.

Resolution Runbook:

  • Verify connectivity to the cloud backend control plane using outbound network debugging assets.
  • Force a real-time parameters refresh directly from an administrative endpoint console window:
# For Linux environments to cycle policy daemon channels
sudo /opt/CrowdStrike/falconctl -v --refresh-policy

Issue: Emergency Sensor Mechanical Uninstallation

When the local asset platform registry or execution drivers break down, standard application software removals will prompt for a Maintenance Token.

Resolution Runbook:

  • Locate the target machine metadata on the cloud engine panel and find its localized Maintenance Token string sequence.
  • Open an elevated administrative terminal instance and execute the manual override extraction payload:
CSUninstall.exe /quiet MaintenanceToken=YOUR_EXTRACTED_MAINTENANCE_TOKEN
